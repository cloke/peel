# Phase 1: Xcode MCP Tool Discovery

**Goal:** Discover all available Xcode MCP tools and create a comprehensive catalog

**Duration:** 1 week  
**Effort:** 3 engineer-days  
**Risk:** Low

---

## Overview

Phase 1 focuses on discovering and documenting all tools exposed by Xcode's mcpbridge. This provides the foundation for Phase 2 implementation.

## What We're Building

### 1. XcodeMCPAdapter Service ✅
**Status:** CREATED  
**File:** `Shared/Services/XcodeMCPAdapter.swift`

Core actor that manages:
- mcpbridge subprocess lifecycle
- STDIO communication with JSON-RPC protocol
- Request/response handling with timeout
- Error handling and logging

**Key features:**
```swift
actor XcodeMCPAdapter {
    func start() async throws
    func listTools() async throws -> [MCPTool]
    func callTool(method: String, toolName: String, arguments: [String: Any]) async throws -> [String: Any]
    func isXcodeAvailable() -> Bool
    func shutdown()
}
```

### 2. Phase 1 Discovery Script ✅
**Status:** CREATED  
**File:** `Tools/xcode-mcp-phase1-discovery.swift`

Executable script that:
1. Verifies Xcode 26.3+ is installed
2. Launches mcpbridge subprocess
3. Sends `tools/list` JSON-RPC request
4. Parses response and organizes tools by category
5. Saves to `tmp/xcode-mcp-tools.json`
6. Generates `Docs/reference/XCODE_MCP_TOOL_REFERENCE.md`

**Run:**
```bash
swift Tools/xcode-mcp-phase1-discovery.swift
```

**Output:**
- `tmp/xcode-mcp-tools.json` — Machine-readable tool catalog
- `Docs/reference/XCODE_MCP_TOOL_REFERENCE.md` — Human-readable reference
